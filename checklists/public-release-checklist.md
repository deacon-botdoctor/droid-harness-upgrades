# Public release checklist

- [ ] No credentials or secret formats.
- [ ] No private hostnames, usernames, IPs, paths, or account routing.
- [ ] No client/private repo names unless public and intended.
- [ ] No raw session transcripts.
- [ ] No provider tokens, env var names, or proxy details that reveal private auth structure.
- [ ] Examples are synthetic.
- [ ] Scripts default to read-only or dry-run where possible.
- [ ] Cleanup/destructive flows require a manifest.
