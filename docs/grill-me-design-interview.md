# Grill me design interview

“Grill me” is a trigger for ruthless planning pressure. Use it before coding when the goal is fuzzy, high-leverage, or easy to overbuild.

## Behavior

- Ask one question at a time.
- Explain why the question matters.
- Provide the recommended answer when possible.
- If the answer can be discovered from the repo, inspect the repo instead of asking.
- Stop when the plan is sharp enough to build.

## Trigger phrases

- “grill me”
- “stress test this”
- “interview me on this plan”
- “make sure this is buildable”

## Output

A build-ready plan with constraints, non-goals, acceptance proof, and first issue slice.
