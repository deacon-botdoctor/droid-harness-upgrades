# Coding harness rules

- Use Compound Engineering for non-trivial work: plan, slice, implement, verify, review, learn.
- When requirements are fuzzy, trigger Grill Me before coding.
- Before ship, run Minimal Implementation Review.
- Prefer reuse, native features, standard library, and installed dependencies before adding new ones.
- Do not call work done without tests, lint, smoke check, or a stated blocker.
- For destructive cleanup, write a cleanup manifest first.
